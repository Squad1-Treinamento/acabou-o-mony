# ADR Placeholder

This document explains what an ADR is and how it is used in this
project. An ADR records a significant technical decision and the reason
it was chosen.

Definition
- A brief record of a decision, context, and consequences.

What it is used for
- to preserve rationale
- to avoid re-litigating past decisions
- to inform future changes

Example
We chose SQLite for local dev to minimize setup time.

Impact on the project
- improves continuity for new contributors
- clarifies constraints for future specs
