# User Story Placeholder

This document explains what a user story is and how it is used in this
project. A user story captures the user's need and the outcome, without
implementation details. It anchors why the work matters.

Definition
- A short statement of value from a user's perspective.
- Focused on intent and acceptance, not design.

What it is used for
- to describe user value and scope
- to set acceptance criteria
- to guide specs and tasks

Example
As a user, I want to reset my password so I can regain access.

Impact on the project
- keeps specs aligned with outcomes
- reduces unnecessary implementation detail early
